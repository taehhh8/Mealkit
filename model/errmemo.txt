리눅스 우분투 --mysql 접속권한 문제

1. 계정생성 및 해당 db사용권한 주기
1) mysql접속 후 계정생성
 $ mysql -u root -p

mysql > create user 'block4s'@'%' identified by 'mypassword';
> flush privileges;

2) db사용권한
mysql> grant all privileges on mealkit.* to block4s@'%';
> flush privileges;


에러 1 : ER_NOT_SUPPORTED_AUTH_MODE: Client does not support authentication protocol requested by server; consider upgrading MySQL client
디버그과정 : mysql> select user, host, plugin from mysql.user;
원인 1 : 패스워드 플러그인 caching_sha2_password를 소화하지 못해서 생기는 오류.
해결 1 : mysql> alter user 'root'@'%' identified with mysql_native_password by 'mypassword';


에러2 : ERROR 1819 (HY000): Your password does not satisfy the current policy requirements
디버그 과정 : 
mysql > SHOW VARIABLES LIKE 'VALIDATE_PASSWORD%';
select user, host, authentication_string from mysql.user;
원인 2 : 패스워드 정책 불충족 
해결 2 : POLICY를 낮춰준다
mysql > set global validate_password_policy = low;
